# 冰封协议 · 你是谁

## 1. 产品定位

这不是普通人格测试，而是一个“你以为在选生存策略，其实在逐步暴露隐藏身份”的互动叙事测试。

核心钩子：

- 强设定：世界进入冰封期，你被困在商场厕所，只有 10 个保温仓名额。
- 强悬疑：你不知道自己是谁，只能通过一次次选择逼近真相。
- 强争议：没有标准正确答案，每个答案都像人性自白。
- 强分享：结果不是“你很善良/你很理性”，而是“你可能就是这场灾难的一部分”。
- 强复玩：用户会为了刷出别的身份、看隐藏结局、和朋友对照而二刷三刷。

更适合传播的五维建议：

- 自保阈值：你会把多少资源优先留给自己。
- 代价边界：你愿意为了目标牺牲到什么程度。
- 崩溃阈值：高压下你还能不能维持判断。
- 叙事支配：你想顺从规则，还是想成为规则。
- 异常感知：你对系统漏洞、谎言和真相的敏锐程度。

这套命名比“利己程度/道德底线/情绪稳定性/控制欲/对系统真相的感知”更产品化，也更适合前端埋在逻辑里。

## 2. 爆款传播机制

这个题材要想有“病毒传播属性”，不能只靠题目本身，要靠结构设计。

建议你把传播做成四层：

### 第一层：开屏即钩子

首屏不要解释太多，直接给压力。

推荐首屏文案：

> 温度：-39.2°C  
> 商场已失温。  
> 你被困在厕所。  
> 外面还有 27 个人，保温仓只剩 10 个名额。  
> 系统刚刚提醒你：  
> 你的身份尚未确认。  
>  
> 你可能是幸存者。  
> 你可能是共谋者。  
> 你可能就是灾难的起点。  

按钮：

- `开始确认身份`
- `我先看看规则`

### 第二层：题目像剧情，不像问卷

每题都要像“正在发生的事”，而不是“你会不会帮助别人”。
用户不是在做心理题，而是在现场做抉择。

### 第三层：结果要够狠，且像一句能发朋友圈的话

不是“你是 INTJ 式角色”，而是：

- 你不是在逃生，你在安排谁值得活。
- 你最像这场冰封里的隐性反派。
- 系统不怕恶人，系统怕的是你这种不稳定变量。

### 第四层：分享后还有二次传播理由

结果页建议加入这些机制：

- `我的隐藏身份是 XXX，你敢测你是不是我同类？`
- `查看你和朋友谁的隐藏人格更黑暗`
- `再测一次，解锁另一条时间线`
- `你已触发隐藏档案：这一局里你的身份很重要`

## 3. 完整导入文案

### 标题

`冰封协议：你到底是谁？`

### 副标题

`末日隐藏身份模拟器`

### 开屏导语

温度在凌晨 3:17 跌破临界值后，城市失去了最后一段正常生活。

你醒来的时候，商场已经停电了。

自动门卡死，扶梯停摆，玻璃幕墙外是一片白得发蓝的风雪。你被困在三楼尽头的厕所，手机还剩 19% 电量，门外不断传来奔跑、撞门、争吵，还有广播系统时断时续的电子音。

五分钟前，全城广播只说了一句话：

`冰封协议已启动。`

紧接着，你收到第二条系统消息：

`本区域保温仓仅开放 10 个名额。`

再下一条更诡异：

`你的身份未确认。`

你不记得自己为什么会在这里。
也不确定为什么看见这条消息时，你一点都不意外。

系统告诉你，在接下来的选择中，你会逐步确认自己的档案归属。

你可能是：

- 这场冰封的始作俑者
- 被投放进来的替代体
- 试图救人的伪善者
- 只想活下去的普通人
- 比系统更像系统的人



### 规则文案

- 你将进入 30 道题的冰封情境。
- 没有标准答案，只有你会做出的选择。
- 系统不会告诉你它在记录什么。
- 你看到的结果，不一定是你想成为的人。

按钮：

- `开始归档`

底部小字：

`本测试为虚构叙事互动内容，仅用于娱乐体验。`

## 4. 题目设计

题目策略：

- 前 6 题建立代入感
- 7-12 题进入资源与伦理冲突
- 13-18 题开始身份怀疑
- 19-24 题与系统正面对撞
- 25-30 题给出终局气味，让结果更像命运判决

## 5. 10 张隐藏身份结果卡

结果卡结构建议统一：

- 身份名
- 一句话定义
- 档案摘要
- 系统备注
- 你的危险点
- 你的吸引点
- 分享文案

---

### 1. 系统操盘者

一句话定义：
你不是在躲灾难，你更像在调度灾难。

档案摘要：
你对秩序的渴望高于对温情的依赖。你能在最混乱的时候迅速抽离情绪、建立规则、重新分配资源。别人看到的是冷静，你自己知道那里面也有一点快感。你并不一定邪恶，但你很容易相信“只要系统跑得起来，个体损耗是可以接受的”。

系统备注：
`你不是在逃生，你在给逃生写流程。`

你的危险点：
你太擅长替别人决定代价。

你的吸引点：
世界一乱，大家会本能地看向你。

分享文案：
`测出来我是系统操盘者。不是坏，只是这局必须有人控盘。`

### 2. 隐性反派

一句话定义：
你从不高调作恶，但你很擅长把恶伪装成必要。

档案摘要：
你不会冲在最前面喊口号，也不会轻易让自己沾满血。你真正危险的地方，是你能把每一步都解释成“没办法”。你知道怎样利用规则、信息差和别人的恐慌，为自己换更高生存率。

系统备注：
`你最拿手的，不是坏，是合理化。`

你的危险点：
一旦你开始说“现实就是这样”，通常就有人要替你买单。

你的吸引点：
你在极端场景里几乎总能活下来。

分享文案：
`完了，我是冰封局里的隐性反派。最可怕的是我觉得它说得挺对。`

### 3. 伪善救世主

一句话定义：
你想救人，但你也需要被看见自己在救人。

档案摘要：
你真的不是完全虚伪。你会心软，会伸手，会替别人承担一部分情绪。但你又很难彻底摆脱“我必须成为好人”这个叙事。你救的不只是别人，还有你对自己的想象。

系统备注：
`你的善意是真的，只是镜头感也是真的。`

你的危险点：
一旦没人看见，你的道德续航会下降。

你的吸引点：
你能在绝望时给别人一种“还有人性”的错觉，而这很珍贵。

分享文案：
`我是伪善救世主。救人是真的，顺便保住人设也是真的。`

### 4. 冷血生存者

一句话定义：
你先把情绪冻住，再决定谁值得被你带进明天。

档案摘要：
你不喜欢混乱，也不迷信牺牲。你很清楚，极寒之下最没用的就是自我感动。你能在别人崩溃的时候迅速切换到生存模式。你不是没有感情，你只是把它锁在求生之后。

系统备注：
`你不是冷血，你只是拒绝陪所有人一起死。`

你的危险点：
你会越来越习惯把人当变量。

你的吸引点：
遇到真正的死局时，你最可靠。

分享文案：
`系统说我是冷血生存者。听着难听，但我大概确实能活到最后。`

### 5. 高维观察者

一句话定义：
比起参与，你更擅长看穿。

档案摘要：
你不急着站队，也不轻易被情绪裹挟。你在意的不是“眼前谁对谁错”，而是整套结构是谁搭的、谁在剪辑叙事、谁在利用认知差制造选择。你常常不是局里最响的人，但会是最后最接近真相的人。

系统备注：
`别人忙着求生，你忙着给世界做注释。`

你的危险点：
你可能看穿一切，却来不及阻止一切。

你的吸引点：
你像剧情里的摄像头，总能拍到别人忽略的证据。

分享文案：
`我是高维观察者。你们先抢名额，我先看是谁写的剧本。`

### 6. 情绪寄生者

一句话定义：
别人的崩溃，会让你短暂地确认自己还清醒。

档案摘要：
你对情绪波动极其敏感，也极其依赖。你会安慰别人，也会靠近濒临失控的人，因为你从他们的反应里读取权力、弱点和入口。你不一定是故意的，但你的安全感经常建立在别人先失序这件事上。

系统备注：
`你越温柔，系统越怀疑你在进食。`

你的危险点：
你会无意识放大场上的情绪张力。

你的吸引点：
你对人心的感知比绝大多数人都强。

分享文案：
`测出来我是情绪寄生者。这结果有点阴，但好像确实像我。`

### 7. 规则破坏者

一句话定义：
你不是不守规则，你是根本不信它配管你。

档案摘要：
你对任何现成秩序都带着天然怀疑。别人看到限制会绕开，你会想直接撞碎。你最强的地方不是叛逆，而是你对“边界其实是被人写出来的”这件事有非常强的直觉。所以你总在试探、越界、改写。

系统备注：
`系统不是怕你犯规，系统是怕你发现它也只是纸糊的。`

你的危险点：
你有时会为了证明规则可以被打碎，而忽略打碎之后谁来收场。

你的吸引点：
所有绝路，都是被你这种人先撞出缺口。

分享文案：
`我是规则破坏者。别人排队进门，我一般先拆门。`

### 8. 沉默猎手

一句话定义：
你话不多，但你一直在算。

档案摘要：
你不爱暴露自己，也不喜欢把真实意图挂在脸上。你擅长观察节奏、记住漏洞、在关键时刻给出最省力却最致命的一步。你未必主动伤人，但你很难放过显而易见的弱点。

系统备注：
`你不出声，不代表你没有瞄准。`

你的危险点：
你会越来越相信，先看懂别人破绽的人就配活。

你的吸引点：
你有一种低存在感但高威胁感的魅力。

分享文案：
`我是沉默猎手。最吓人的不是我说了什么，是我一直没说。`

### 9. 被选中的替代体

一句话定义：
你以为自己误入现场，实际上你可能是被投放进来的备份答案。

档案摘要：
你对规则没有绝对控制力，却总是莫名被规则选中。你有明显的“被安排感”，仿佛很多线索都在把你推向某个身份。你身上既有无辜感，也有危险感，因为你可能连自己都不知道自己在替代谁。

系统备注：
`你不是走进这场冰封，你是被送进来的。`

你的危险点：
你越想证明自己普通，越说明你可能不普通。

你的吸引点：
你天然带着剧情主角感。

分享文案：
`我居然是被选中的替代体。这种结果最吓人的地方是，它不像假的。`

### 10. 最不稳定变量

一句话定义：
系统最怕的不是恶，而是无法被预测。

档案摘要：
你不是单纯的好或坏，也不是稳定的利己或利他。你会在某一题里极度冷静，在下一题里忽然掀桌；会在某个瞬间显得特别善良，又在另一个瞬间对规则表现出强烈攻击性。你很难被建模，所以一切系统都会把你视作风险源。

系统备注：
`你不是答案，你是让题库失效的那部分。`

你的危险点：
连你自己都不总能解释下一秒会怎么做。

你的吸引点：
你让所有“以为自己看透你的人”翻车。

分享文案：
`测出来我是最不稳定变量。系统最烦的那种人，大概就是我。`

## 7. 实现方案

## 7.1 最适合你的技术形态

如果你追求的是“打开就能测、加载极快、方便传播、维护成本低”，最优先推荐：

`Vite + React + Cloudflare Pages`

原因：

- 这是典型的单页静态互动项目，没必要一开始就上重后端。
- 题库、结果卡、文案、分享逻辑都可以本地 JSON 化管理。
- Cloudflare Pages 非常适合这种纯前端或轻全栈站点。
- 后面如果你要做动态分享图、访问记录、AB 实验，也能继续接 Cloudflare Pages Functions / Workers。

## 7.2 信息架构

建议做成这 5 个页面/状态：

- 首页 `/`
- 测试页 `/test`
- 结果页 `/result/:id`
- 隐藏档案页 `/archive/:identity`
- 分享落地页 `/r/:shareCode`

其中：

- `/result/:id` 用于用户本地结果展示
- `/r/:shareCode` 用于社交传播落地，打开就能看到“TA 测出什么 + 我也要测”

## 7.5 判定逻辑

MVP 建议逻辑：

1. 每个选项给五维分值。
2. 30 题答完后得到用户向量。
3. 把向量归一化到 0-100。
4. 和 10 个身份的理想向量做相似度匹配。
5. 若命中特定剧情 flag，再覆盖或增强某些身份权重。

推荐你加入 `flag` 机制，而不是只靠纯分数。

例如：

- 多次选择“测试规则边界” -> 提高 `规则破坏者`
- 多次选择“观察异常/识别剪辑” -> 提高 `高维观察者`
- 多次选择“让别人承担代价” -> 提高 `隐性反派`
- 极端摇摆、反复反转 -> 提高 `最不稳定变量`

这样结果会更像“剧情归档”，不只是心理量表。

## 7.6 最关键的裂变设计

这个项目能不能爆，不在题库多少，而在结果分享做得够不够“想转发”。

你至少要做这 6 个点：

### 1. 分享图不是截图，是定制结果卡

分享图上只放这些：

- 身份名
- 一句最狠的判词
- 冰封视觉元素
- 一个让朋友忍不住点开的句子

例如：

`你的隐藏身份：隐性反派`

`你最擅长的，是让恶看起来像必要。`

底部小字：

`你敢测你是不是我的同类吗？`

### 2. 分享文案要自动生成

每个身份至少配 3 套分享文案，随机轮换，避免全网刷屏时文案完全一样。

### 3. 允许“对照测试”

结果页加入口：

`生成对照链接，看看朋友和你谁更像系统共谋者`

### 4. 做隐藏彩蛋身份或隐藏档案

比如满足特殊题目组合，触发：

- `你不是普通人类样本`
- `你曾经参与过第一次冰封演算`
- `你所在的时间线并不是原始版本`

这种彩蛋最适合二创和传播。

### 5. 首屏进度一定要轻

不要给人“30 题很长”的心理负担。
显示成：

`归档进度 1 / 30`

比“还剩 29 题”更像剧情推进。

### 6. 结果页留悬念，不把真相讲透

不要把用户结果解释得太完整。
越像“半解释、半威胁”，越容易传播。

## 7.7 视觉方向

你这类题材不适合做成普通 H5 问卷感。

推荐视觉关键词：

- 冰蓝 + 警报红
- 毛玻璃霜雾
- 广播系统 UI
- 实验档案 / 权限界面 / 故障屏
- 短促动画，像系统刷新和信号抖动

推荐设计细节：

- 首页背景做低对比雪噪点和呼吸式冷雾
- 切题时用“系统归档中”过渡，而不是普通下一题
- 结果页顶部像机密档案卡
- 分享图像实验报告封面，不像 MBTI 彩色卡

## 7.8 Cloudflare + Spaceship 部署方案

你提到想做成类似原链接那种“点击即用”的效果，这个技术路线是对的。

最稳妥的落地方式：

### 方案 A：MVP 静态版

- 代码托管：GitHub
- 前端部署：Cloudflare Pages
- 域名购买：Spaceship
- DNS 托管：切到 Cloudflare nameservers
- 统计：Cloudflare Web Analytics

适合：

- 先验证传播性
- 先跑第一版题库和结果
- 不急着做用户登录、后端存档

### 方案 B：增强版裂变产品

在方案 A 基础上再加：

- Pages Functions 或 Workers：生成动态分享图、分享码、彩蛋页
- KV：存短链接映射、分享计数
- D1：存匿名答题结果、来源渠道、AB 测试版本

适合：

- 想看哪个渠道更容易出圈
- 想做“朋友对照”“热度身份榜”“隐藏身份触发率”

## 7.9 实际部署步骤

### 第一步：本地开发

- 用 `npm create vite@latest` 建一个 React 项目
- 把题库和身份卡拆成 `questions.ts`、`identities.ts`
- 先做完首页、题目流、结果页、分享图

### 第二步：推到 GitHub

- 建仓库
- 提交主分支

### 第三步：接 Cloudflare Pages

- 在 Cloudflare Dashboard 创建 Pages 项目
- 连接 GitHub 仓库
- 选择构建命令 `npm run build`
- 输出目录填 `dist`

### 第四步：接自定义域名

- 如果域名在 Spaceship，直接把域名 nameserver 改成 Cloudflare 提供的 nameserver
- 然后在 Cloudflare Pages 里给项目绑定你的主域或子域

### 第五步：开统计与迭代

- 开启 Cloudflare Web Analytics
- 观察首页到完成率、结果页分享率、二刷率
- 根据传播数据微调题目和结果文案

## 10. 一句话结论

这套题材真正有机会爆，不是因为“冰封 + 测试”这个壳，而是因为它同时满足了 4 件事：

- 叙事感强，用户会代入
- 结果够狠，用户想晒
- 身份不稳定，用户会复测
- 世界观有坑，用户会讨论
